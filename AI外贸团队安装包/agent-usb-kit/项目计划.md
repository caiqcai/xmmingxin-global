# 阿里国际站客户智能体安装包 — 项目计划

## 项目目标

为阿里国际站的客户（服务中 / 新签 / 潜在）制作一套 U 盘安装包，实现：
- 安装时选行业 → 注入对应行业知识
- 安装时收集客户信息 → 个性化首次体验
- 客户打开即用 → 30秒内感受价值 → 推动付费

---

## 一、智能体设计（3个角色）

### 任务 1.1：设计「产品经理」智能体
- **职能定位**：选品 + 1688找货 + 发品 + 详情页设计 + 店铺装修
- **交付物**：SOUL.md + 精选技能清单（10-15个）
- **技能来源**：从现有「选品大师」「发品牛马」「结构化商详设计师」「店铺装修设计师」中提取
- **优先级**：P0
- **预计工作量**：30分钟

### 任务 1.2：设计「运营总监」智能体
- **职能定位**：店铺数据分析 + 流量监控 + SEO优化 + 营销策略 + A/B测试
- **交付物**：SOUL.md + 精选技能清单（10-15个）
- **技能来源**：从现有「运营大师」中提取
- **优先级**：P0
- **预计工作量**：30分钟

### 任务 1.3：设计「业务员」智能体
- **职能定位**：客户开发 + 企业背调 + 邮件营销 + 谈判策略 + 询盘回复
- **交付物**：SOUL.md + 精选技能清单（10-15个）
- **技能来源**：从现有「客户开发大师」「背调专员」「客户营销大师」「谈判军事」中提取
- **优先级**：P0
- **预计工作量**：30分钟

### 任务 1.4：确认技能分配无遗漏无冲突
- **内容**：列出三个智能体的完整技能对照表，确认没有重要技能被遗漏，没有不相关的技能被误装
- **交付物**：技能分配对照表
- **优先级**：P0
- **预计工作量**：15分钟

## 二、首次体验设计（FTUE）

### 任务 2.1：设计安装时的信息收集流程
安装脚本中依次收集以下信息：
1. **称呼** — "老板，我应该怎样称呼您？" → 写入 USER.md
2. **主营产品** — "您的公司主要做什么产品？" → 写入 USER.md
3. **目标市场** — "主要出口哪些国家/地区？" → 写入 USER.md
4. **是否有团队** — "您现在有业务团队吗？" → 影响首条消息话术
5. **行业选择** — 36个一级类目菜单 → 决定注入哪个 MEMORY.md

- **交付物**：安装脚本中的交互流程设计文档
- **优先级**：P0
- **预计工作量**：20分钟

### 任务 2.2：设计首条消息模板
每个智能体的首条消息包含以下模块：

```
[1] 自我介绍（我是谁、我能做什么）
[2] 吉祥话（区分有团队 / 无团队两个版本）
    - 有团队版："帮您和团队一起提效，让每个人都如虎添翼"
    - 无团队版："从今天起我就是您的团队，选品发品找客户都交给我"
[3] 数字选择菜单（3个预设提示词 + 1个自由提问，用客户的主营产品生成）
```

- **交付物**：3个智能体 × 2个版本（有团队/无团队）= 6套首条消息模板
- **优先级**：P0
- **预计工作量**：40分钟

### 任务 2.3：设计体验后的定时任务引导
客户完成第一次体验后，智能体主动追问是否设定定时任务：

| 智能体 | 推荐定时任务 | 频率 |
|--------|-------------|------|
| 产品经理 | 行业竞品动态 + 新品机会报告 | 每周一 |
| 运营总监 | 店铺流量周报 + 异常预警 | 每周一 |
| 业务员 | 新采购商推荐 + 询盘跟进提醒 | 每日 |

- **交付物**：定时任务引导话术模板 + cron 任务配置模板
- **优先级**：P1（首条消息做好后再做）
- **预计工作量**：30分钟

## 三、行业知识包（MEMORY）

### 任务 3.1：确定行业类目列表
- **内容**：基于阿里国际站官方36个一级类目，去掉重复和不适用的，确定最终列表
- **交付物**：行业类目清单（预计30-33个）
- **优先级**：P0
- **预计工作量**：15分钟

### 任务 3.2：设计 MEMORY.md 标准模板
每个行业知识包的统一结构：
```
# 行业知识 — {行业名}

## 核心品类方向（含主要二级品类）
## 买家画像（主力市场、采购特征、决策周期）
## 关键认证与合规要求（按市场分）
## 定价参考（采购价→报价的倍率逻辑）
## 发品关键词库（核心词 + 长尾词 + 避坑词）
## 旺季节奏（按月份）
## 行业避坑指南（常见翻车点）
```

- **交付物**：MEMORY.md 模板文件
- **优先级**：P0
- **预计工作量**：15分钟

### 任务 3.3：批量编写行业知识包
- **内容**：按模板为每个行业编写 MEMORY.md，需结合搜索获取最新的行业数据
- **交付物**：30-33个行业的 MEMORY.md 文件，存放在 `industry-packs/{行业名}/MEMORY.md`
- **优先级**：P0
- **预计工作量**：较大，预计 3-4 小时（可并行编写）
- **质量要求**：每个行业的关键词、认证、热销品类需要有真实数据支撑，不能凭空编造

## 四、安装脚本开发

### 任务 4.1：改造 install.sh 主脚本
在现有安装脚本基础上增加：
1. 行业选择菜单（一级类目列表）
2. 客户信息收集（称呼/主营/市场/团队）
3. 根据选择注入对应行业的 MEMORY.md
4. 根据收集的信息生成 USER.md
5. 根据角色 + 客户信息 + 团队情况生成首条消息写入 BOOTSTRAP.md
6. 安装 3 个智能体（产品经理 + 运营总监 + 业务员），一次安装全部到位

- **交付物**：完整可运行的 install.sh
- **优先级**：P0
- **预计工作量**：1小时

### 任务 4.2：改造 export.sh 导出脚本
- **内容**：导出脚本需要把 3 个智能体 + 全部行业知识包一起打包成一个 zip
- **交付物**：完整可运行的 export.sh
- **优先级**：P1
- **预计工作量**：20分钟

### 任务 4.3：更新 README.md 使用说明
- **内容**：更新说明文档，反映新的安装流程
- **优先级**：P1
- **预计工作量**：15分钟

- **优先级**：P0

### 任务 5.2：最终打包

- **交付物**：可直接拷贝到 U 盘的 zip 文件
- **优先级**：P0
- **预计工作量**：5分钟
